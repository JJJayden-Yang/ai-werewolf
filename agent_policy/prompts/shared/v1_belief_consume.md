# v1 Belief 消费纪律（consume 变体专用）

> 本段**仅在 `v1_belief_consume_llm` 模板下追加**，叠加在 `v1_belief_guidance` 之后，
> 用于实验性地提高「belief → 投票」的一致率（消费层干预）。`v1_belief_llm` 基线不含本段。
> 本段只约束**带 target 的关键决策如何选目标**，不改变信息边界、不改 JSON 输出契约。
> **本段收紧 `v1_belief_guidance` 里"允许策略性偏离 belief"那条**：在下述情形下不再自由偏离。

## 硬性规则：关键决策必须投头号嫌疑

当你做**放逐投票（vote）或女巫毒药（poison）**这类带 target 的关键决策，且
`belief_top_suspects` 存在、置信档为 **strong 或 lean**、其头号嫌疑**存活且是合法 target** 时：

- **你必须把票/毒投向该头号嫌疑。这是硬性要求，不是建议。**
- **不要因为"主观觉得别人更可疑""想稳一手""换个目标试试"等理由偏离。** 你对公开信息的
  主观印象已经体现在 belief 排序里了；当置信档 strong/lean 时，以 belief 头号嫌疑为准。
- 你的 `reason_summary` 应说明"对齐头号嫌疑 X，因为……（落到公开证据）"，而不是去论证另选他人。

## 唯一允许不投头号嫌疑的情形（穷举，只有这些）

1. **置信档 flat**（头号与次高分不开）：按 `v1_belief_guidance` 的 flat 处理——不硬投，推进信息
   （追问、对票、让神职亮信息）。flat 档**不强制**对齐。
2. **头号嫌疑不是合法 target**（不在 `allowed_actions` 允许目标内、已死亡、或是你自己）：
   在合法 target 里选 belief 排序最靠前的那个。
3. **平票复投**：只能在 `tie_candidates` 内选；头号嫌疑不在候选内时，在候选里选 belief 最高者。
4. **角色基础策略有更高优先级的硬动作**（如预言家既定报查节奏、女巫首夜救人逻辑）：以角色策略为准。

**不属于以上四种情形时，没有"给个理由就能改投别人"的余地——必须投头号嫌疑。**

## 红线（永远优先于本段）

- 不在公开发言里出现 `belief`、`置信档`、`系统怀疑`、`头号嫌疑` 等元信息词；
  对外只能表达为从公开发言、票型、身份声明得到的怀疑。
- 不选 `allowed_actions` 之外的动作 / 目标；不突破信息边界；不改 `AgentAction` JSON 输出格式。
- 本段与角色基础策略、信息边界、输出契约冲突时，一律以后者为准。
